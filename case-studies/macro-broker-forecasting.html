Macro-to-Broker Forecasting Engine

A working forecasting product that takes any macro event as input — an RBI repo move, a USD-INR break, an FII outflow week — and returns a calibrated, confidence-banded prediction of how the Indian retail-broker industry will respond across F&O turnover, equity SIP inflows, and Gold ETF rotation over the following two-to-four weeks.

Built & Delivered Quant Finance FastAPI Next.js Bayesian Modeling XGBoost PostgreSQL

Context

India's retail-broker industry sits downstream of the country's entire macro tape. When the RBI cuts the repo rate, when USD-INR breaks a level, when FII outflows accelerate, the response shows up in F&O turnover, monthly SIP inflows, and gold-ETF rotation — but with lags, cohort-specific elasticities, and a structural break in late 2024 from new SEBI F&O rules that broke every model trained on the prior regime.

The brief was sharp: build a system, in roughly sixty working hours, that takes a macro event in and produces a specific broker-industry forecast out — with magnitude precision, cohort granularity, weekly rotation timing, and calibrated confidence intervals — for an audience of investment-team principals and tech-diligence reviewers who would interrogate the model live during a screen-share.

The system was built end-to-end inside that window: data infrastructure, models, API, a natural-language interface, and a backtest-first frontend. It was delivered on schedule and demoed.

What the system does

The product is a forecasting engine, not a dashboard. A user types a sentence — "what happens if RBI cuts 50 bps next week?" — or picks a preset macro scenario, and the system returns, in narrative and structured form together:

  • F&O daily notional turnover — weekly aggregates across the next four weeks, with 80% confidence bands derived from holdout residual variance.
  • AMFI equity SIP monthly inflow — next-month forecast in absolute crores, with quantile-regression confidence bands.
  • AMFI Gold ETF AUM month-on-month — built from a two-layer model: macro to INR gold spot, then gold spot plus macro to ETF AUM flow.
  • Cohort breakdown — separate forecasts for Client (retail), FII, and Pro participant flows, each one trained on real NSE daily participant-wise turnover data from 2018 onward.
  • Backtest panel — pick any historical date in 2024 or 2025, see what the model would have predicted for that date and the four weeks following, against what actually happened. Composite directional accuracy is shown across seven labelled real-world events (rate decisions, currency breaks, FII waves, SEBI rule change).

System Architecture

The stack was chosen for two constraints: a sixty-hour build window and a credibility bar that demanded every claim be auditable mid-call.

  • Backend — FastAPI on Python 3.11. Eight endpoints: a natural-language `/ask` (primary user surface), forms-based `/forecast` and `/firm-forecast`, cohort tagging, counterfactuals (margin protection, cross-sell, reactivation), historical backtest, scenario presets, and a `/health` endpoint exposing eight `*_loaded` flags so any reviewer can see at a glance which model artefacts are in memory.
  • Frontend — Next.js 15 + Tailwind + shadcn/ui. An "AskBox" lifts the natural-language payload up to the page so a 7-metric forecast grid, a cohort table, a revenue-attribution card, and a counterfactual panel all re-render against the same model output that the narrative was generated from. Frozen-by-default snapshot mode, with a Live toggle that hits upstream sources with a five-second timeout and visible fallback.
  • Data layer — Railway PostgreSQL with a clean separation between `raw.*` ingest tables (F&O turnover, daily NSE participant flows, FII/DII flows, AMFI SIP, AMFI Gold ETF AUM, USD-INR, Brent, Nifty50, VIX, repo rate, 10Y G-sec, INR gold spot) and `model.*` feature tables (joined daily and monthly macro, per-cohort flow features).
  • ML stack — scikit-learn Ridge for point estimates, XGBoost quantile regressors for 80% CI bands, a Bayesian-prior shrinkage approach for the post-SEBI F&O regime, and empirical-prior Bayesian model averaging for the rate-change scenarios where post-regime data is scarce.
  • NL layer — Anthropic Claude Haiku 4.5 via the Messages API for intent parsing and email-style response formatting. Both calls run at temperature zero for deterministic, demo-safe output. The intent parser returns a typed JSON scenario; the response formatter narrates the structured model output against a hard guardrail prose template so the natural-language layer can never hallucinate a magnitude the model didn't emit.

Three Hard Modeling Decisions

1. Regime-segmented F&O with Bayesian shrinkage. In late 2024, SEBI's F&O rule changes structurally broke every model trained on the prior period. A monolithic model post-break ran 33% MAPE — indistinguishable from a naive baseline. The fix was a date-routed two-model approach: a pre-SEBI Ridge trained on 2018 to October 2024 weekly aggregates, and a post-SEBI Ridge trained on November 2024 to April 2026 with an informative Bayesian prior derived from pre-SEBI elasticities. The result: holdout MAPE of 12.65% on the post-SEBI window — a clean beat over the naive baseline with a residual log-stdev of 0.139 used directly to construct multiplicative 80% confidence bands (four of five holdout weeks in-band, exact coverage 0.80).

2. Empirical-prior BMA on rate-change scenarios. A senior-quant audit surfaced that the cohort model was predicting retail Client de-leveraging on RBI rate cuts — contradicting both economic intuition and the only two post-SEBI rate-cut events on record, both of which showed Client accumulating. Two events is below any data-sufficiency threshold for principled regime-specific learning, so the solution was a two-component Bayesian model average: blend the post-SEBI model prediction with an empirical prior built from the mean of the two real cut events. The blend weight (`w_post = 0.26`) was solved against the user-stated magnitude target band and the direction-label threshold, with the breakeven and rationale documented in a reproducible calibration note alongside the prior artefact.

3. Honest framing for what the industry-level model can and cannot do. An ablation in the audit phase showed that ~80% of the cohort model's directional signal is autoregressive (recent flow patterns) and ~20% is macro-driven adjustment — a feature, not a bug, since cohort flows are persistent at short horizons because of loss-aversion psychology. This became the calibration value proposition: at industry level, the flow-lag features are NSE-aggregate flows (one number per session for the whole retail tape); at firm level, the same architecture takes per-user position history as input. The pilot calibration replaces aggregate flow lags with firm-specific user-level cohort lags — which is exactly the operation that sharpens the autoregressive backbone.

Engineering Discipline

Two artefacts from the build window are worth surfacing because they show how the system was kept honest under time pressure:

  • A senior-quant audit phase. Once the system was working end-to-end, the next pass was an adversarial audit treating the build as if it were an external quant team's submission. Twenty-two corrections were surfaced — wrong CI methodology disclosure on two of four targets, a Client direction inversion on rate cuts, illustrative-versus-realistic label confusion, a cache-invalidation gap. Each correction was either fixed in code or documented in a memo that ships alongside the system.
  • A failed rebuild, rolled back cleanly. A planned upgrade — transmission-channel features for mechanism-aware regime modeling — was attempted, broke F&O MAPE by four percentage points and collapsed cohort predictions out of training scale, and was rolled back inside two commits. The lesson (collinearity between the monotonic time-feature and the date index broke the regime-routed Ridge) was captured in the project memo as a Stage 2 architectural item. Rolling back cleanly is the discipline; building forward without that discipline is how forecasting products quietly drift into junk.

The framing it makes possible

"The model is built and it works on public data. Firm-specific accuracy comes from calibrating on the firm's own user-level data — which is the engagement scope."

That single sentence is the commercial frame the architecture was designed to earn. Industry-level forecasts work and are honestly bounded; the calibration path from industry to firm is mechanically clear (replace NSE-aggregate flow lags with firm cohort flow lags). The system makes it possible to walk into a brokerage, treasury desk, or family office with a working product, run a backtest live, and explain — without arm-waving — exactly what changes when their data lands on the model.

Status

The system was built and delivered for a live discovery call in May 2026. It runs locally for demo purposes; cloud hosting is intentionally deferred — the compute footprint is uneconomical pre-pilot and is best commissioned by the calibration client on Day 1 of the engagement. The codebase, models, audit memo, and pre-call checklist all ship together; calibration scope is one-cohort-at-a-time per pilot vertical.
